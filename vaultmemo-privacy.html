Privacy Policy

App: VaultMemo v1.2  ·  Developer: no10messi.skr  ·  Last updated: March 2026

VaultMemo is a local-first, privacy-focused encrypted memo app. We do not operate any proprietary servers. Your memo content is never transmitted to any server or recorded on the blockchain. Only a cryptographic hash fingerprint and transaction metadata are recorded on-chain.

Data Collection

VaultMemo collects no personal information. No account or registration is required.

Local Data Storage

The following data is stored locally on your device only and never transmitted to any server:

  • Wallet info: Your public key (wallet address), MWA auth token, and wallet URI base for reconnecting without re-authorization.
  • Encryption key: Stored in the Android Keystore (TEE), protected by biometric authentication. Never exported or transmitted.
  • Transaction metadata: TxID, SKR amount, and timestamp are embedded inside each .vam file and cached in AsyncStorage. Never transmitted to any server.
  • Encrypted memo files (.vam): Stored in the app's internal DocumentDir/VaultMemo/ folder (for in-app file list) and copied to Downloads/VaultMemo/ via MediaStore (for permanent storage accessible via file manager).
  • Theme preference: Light/dark mode setting, stored locally.

In-App File Management (v1.2)

VaultMemo v1.2 includes an in-app file list that reads encrypted .vam files stored in DocumentDir/VaultMemo/. All file operations (list, rename, delete) are performed locally on-device only. No file contents are transmitted to any external service during these operations.

  • File list: VaultMemo reads file metadata (name, size, modification date) from DocumentDir/VaultMemo/ to display the file list. No file contents are read unless the user initiates decryption.
  • Delete: Files are permanently deleted from DocumentDir/VaultMemo/ only. Files in Downloads/VaultMemo/ (MediaStore) are not affected.
  • Rename: File names are changed locally within DocumentDir/VaultMemo/. No external services are contacted.

On-Chain Integrity Verification (v1.2)

When you tap "Verify On-Chain" on a decrypted file, VaultMemo:

  • Sends a request to a Solana RPC endpoint to fetch the transaction by its TxID. VaultMemo uses a three-endpoint fallback chain in the following order: rpc.ankr.com/solana (primary), api.mainnet-beta.solana.com (secondary), mainnet.helius-rpc.com (last resort). Only the TxID is transmitted; no memo content or private data is sent.
  • Parses the transaction's log messages to extract the stored hash prefix.
  • Computes a local SHA-256 hash of the encrypted file content and compares the first 16 hex characters with the on-chain value.
  • No memo content or private data is transmitted during this process — only the TxID is sent to the RPC endpoint.

Encryption Certificate (v1.2)

VaultMemo v1.2 allows users to generate a shareable encryption certificate image. The certificate image is generated entirely on-device and contains only:

  • File name (as displayed in the app)
  • Wallet address (truncated: first 6 and last 6 characters)
  • Transaction ID (truncated)
  • Encryption tier, algorithm, file size, and timestamp

Certificate images are generated locally using react-native-view-shot. Sharing is performed through the Android system share sheet. No certificate data is transmitted to VaultMemo servers (none exist).

File Storage (On-Device)

Upon user action, VaultMemo writes encrypted .vam files to two locations:

  • DocumentDir/VaultMemo/ — for in-app file list access (cleared if app is uninstalled)
  • Downloads/VaultMemo/ via MediaStore — for permanent storage accessible via file manager (persists after app uninstall)

Each file contains the AES-256-GCM encrypted memo (base64), the Solana TxID, and a timestamp. Memo content is never readable without the correct encryption key. These files are not transmitted to any server.

Blockchain Data (Public)

When you save a memo, VaultMemo submits a Solana transaction containing:

  • A partial hash: the first 16 hex characters of the SHA-256 hash of your encrypted data, plus a Unix timestamp (SPL Memo instruction).
  • An SKR token transfer from your wallet to the developer's wallet (usage fee — see pricing tiers in the EULA).
  • A Solana Pay reference key (randomly generated) for transaction tracking.

Your wallet address and the above data are visible on the public Solana blockchain. Your memo content is never included in or derivable from the on-chain record.

Biometric & Hardware Security

Biometric verification is handled entirely by the Android OS and TEE. The app never receives raw biometric data. The encryption key is invalidated if the device fingerprint is changed.

Wallet Security

VaultMemo does not access your private keys or seed phrase. All signing is handled by the Solana Mobile Seed Vault via the Mobile Wallet Adapter (MWA) protocol. Wallet authorization is restricted exclusively to the Seed Vault; third-party wallet apps are blocked at the application level.

Third-Party Services

VaultMemo does not integrate advertising, analytics, or tracking services. It communicates with the following external services for blockchain operations and price data only:

  • Solana RPC (fallback chain): rpc.ankr.com/solana (primary), api.mainnet-beta.solana.com (secondary), mainnet.helius-rpc.com (last resort) — transaction submission, balance queries, and on-chain verification. Only your public wallet address and TxID are included. The first available endpoint is used; subsequent endpoints are tried only if the previous one fails.
  • Jupiter Price API (api.jup.ag): Real-time SKR/USD price for fee calculation.
  • CoinGecko API (api.coingecko.com): Fallback price source.
  • Firebase Remote Config (Google): Used to deliver app announcements and dynamic pricing configuration. Firebase Remote Config may collect a Firebase Installation ID and basic device/app metadata (OS version, app version) as part of its standard operation. This data is processed by Google under their Privacy Policy. No memo content or wallet private keys are transmitted. You can review Firebase's data practices at firebase.google.com/support/privacy.
  • Solscan (solscan.io): Opened in your browser when you tap a TxID. No data is sent by the app itself.

Children's Privacy

VaultMemo does not knowingly collect any data from anyone, including children under 13.
