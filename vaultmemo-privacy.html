Privacy Policy

App: VaultMemo v1.0  ·  Developer: no10messi.skr  ·  Last updated: March 2026

VaultMemo is a local-first, privacy-focused encrypted memo app. We do not operate any proprietary servers. Your memo content is never transmitted to any server or recorded on the blockchain. Only a cryptographic hash reference and transaction metadata are recorded on-chain.

Overview

VaultMemo allows users to encrypt personal memos using a key derived from their Solana Seed Vault wallet, store encrypted files locally on their device, and record a partial integrity hash on the Solana blockchain. The app never holds, reads, or transmits the content of your memos to any external party.

Data Collection

VaultMemo collects no personal information. We do not collect your name, email address, phone number, location, or any other identifying information. No account or registration is required to use the app.

Local Data Storage

VaultMemo stores the following data locally on your device only. This data never leaves your device and is not transmitted to any server:

  • Wallet connection info: Your public key (wallet address), MWA auth token, and wallet URI base for reconnecting to the Seed Vault without re-authorization.
  • Encryption key: Your AES-256 encryption key is stored in the Android Keystore (TEE hardware), protected by biometric authentication. It is never exported or transmitted.
  • Transaction metadata: For each saved memo, the corresponding Solana transaction ID (TxID), approximate SKR amount paid, and timestamp are embedded directly inside the encrypted .enc file itself. This data is also cached in AsyncStorage for quick access. It is never transmitted to any server.
  • Theme preference: Your selected UI theme (light/dark) is stored locally.

File Storage (On-Device)

VaultMemo writes the following files to your device storage upon user action. These files are not transmitted to any server:

  • Encrypted memo files (.enc): Each file is a JSON document containing the AES-256-GCM encrypted memo (base64), the Solana transaction ID (TxID), and a timestamp. Files are saved to Downloads/VaultMemo/ and the app's internal document directory (DocumentDir/VaultMemo/). The memo content is never readable without the correct encryption key, and the TxID embedded in the file allows verification of the corresponding on-chain record.

These files remain entirely on your device. You can delete them at any time using your device's file manager. Without a backup, deleted files are permanently lost.

Blockchain Data (Public)

When you save a memo, VaultMemo submits a Solana transaction that contains:

  • A partial hash reference: the first 16 hexadecimal characters of the SHA-256 hash of your encrypted memo data, plus a Unix timestamp. This is recorded in an SPL Memo instruction.
  • An SKR token transfer from your wallet to the developer's wallet address as a usage fee (~$0.75).
  • A Solana Pay reference key (a randomly generated public key) for transaction tracking purposes.

Your wallet address (public key) and the above transaction data are visible on the public Solana blockchain. Your memo content itself is never included in or derivable from this on-chain record.

Biometric & Hardware Security

VaultMemo requires fingerprint authentication to unlock the vault. Biometric verification is handled entirely by the Android OS and the device's Trusted Execution Environment (TEE). The app never receives raw biometric data. The encryption key is bound to the currently registered fingerprint; if the fingerprint is changed, the key is automatically invalidated and must be re-derived.

Private Keys & Wallet Security

VaultMemo does not access your private keys or seed phrase. All cryptographic signing is handled exclusively by the Solana Mobile Seed Vault. VaultMemo communicates with the Seed Vault through the Mobile Wallet Adapter (MWA) protocol, which never exposes private key material to the application. Wallet authorization is restricted exclusively to the Seed Vault; third-party wallet apps (e.g., Phantom, Solflare) are blocked at the application level.

Third-Party Services

VaultMemo does not integrate any advertising, analytics, or tracking services. The app communicates with the following external services solely for blockchain operations and price data:

  • Solana RPC endpoints: api.mainnet-beta.solana.com (primary) and mainnet.helius-rpc.com (fallback) for submitting transactions and fetching wallet balances. Only your public wallet address is included in these requests. No personal data is transmitted.
  • Jupiter Price API (api.jup.ag): Primary source for real-time SKR/USD price data used to calculate the memo save fee.
  • CoinGecko API (api.coingecko.com): Fallback price data source used only when Jupiter is unavailable.
  • Solscan (solscan.io): When you tap a transaction ID in the app, your device's browser opens the corresponding Solscan page. No data is transmitted by the app itself.

No personal data is transmitted to any of these services. Price data displayed in the app is approximate and may differ from actual on-chain values.

Children's Privacy

VaultMemo does not knowingly collect any data from anyone, including children under the age of 13.
