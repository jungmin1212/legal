Security Restrictions

VaultMemo enforces the following security measures at the application level:

  • Seed Vault only: Wallet authorization is restricted exclusively to the Solana Mobile Seed Vault. Connections from third-party wallet apps (e.g., Phantom, Solflare) are blocked. This is a deliberate design decision to prevent unauthorized key derivation and transaction signing.
  • Hardware-bound encryption key: Your AES-256 encryption key is stored in the Android Keystore (Trusted Execution Environment) and bound to your currently registered fingerprint. The key cannot be extracted from the device.
  • Session lock: The encryption key is cleared from memory whenever the app enters the background or the screen is turned off. Biometric re-authentication is required to resume.

These restrictions are non-configurable and exist solely for user protection.

Payment Terms

Each memo save operation requires a one-time payment of approximately $0.75 USD in SKR (Seeker Rewards Token) on the Solana blockchain. This fee covers the on-chain hash recording via Solana Pay. The exact SKR amount is calculated at the time of saving based on the real-time SKR/USD market price fetched from Jupiter Price API or CoinGecko.

  • All SKR payments are processed directly on the Solana blockchain and are irreversible once confirmed.
  • The developer is not responsible for any losses arising from price volatility between the time of price display and transaction confirmation.
  • No refunds are issued for confirmed on-chain transactions.
  • The user is responsible for maintaining sufficient SKR balance before initiating a save operation.

Storage Permissions

VaultMemo requests storage permissions for the following purposes:

  • Encrypted file storage: Saving .enc files to Downloads/VaultMemo/ on your device when you tap "Encrypt & Save".
  • Internal document directory: Maintaining a file listing in the app's internal storage (DocumentDir/VaultMemo/) for browse functionality. This directory is cleared when the app is uninstalled.

Storage is accessed only while the app is in the foreground. No files are written by background services.

Data Loss Disclaimer

Encrypted .enc files saved to device storage are the user's sole responsibility. The developer is not liable for data loss resulting from:

  • Device failure, loss, or damage
  • App uninstallation: Clears the internal document directory. However, encrypted .enc files saved to the public Downloads/VaultMemo/ folder will persist on your device. You can manually re-import these files after reinstallation.
  • Loss of the Seed Vault seed phrase (renders decryption permanently impossible on any device)
  • Fingerprint change without prior re-derivation of the encryption key
  • User deletion of backup files

It is strongly recommended to maintain encrypted file backups in multiple locations (email, cloud storage, external drive).
