Security Architecture

  • Seed Vault exclusive: VaultMemo is restricted exclusively to the Solana Mobile Seed Vault. Third-party wallet apps are blocked at the application level.

  • Session lock: The encryption session is cleared whenever the app enters the background. Biometric re-authentication is required to resume.

In-App File Management

VaultMemo v1.2 includes the following local file management features. All operations are performed on-device only and do not require additional payment:

  • File list: View all encrypted .vam files saved in the app's internal storage, with search, sort by date/size/name.
  • Decrypt from list: Tap any listed file to decrypt and view its contents. Biometric authentication is required.
  • Rename: Rename any .vam file within the app's internal storage. Free of charge.
  • Delete: Permanently delete a .vam file from the app's internal storage. This action is irreversible. Files in Downloads/VaultMemo/ are not affected by in-app deletion.

On-Chain Integrity Verification

VaultMemo v1.2 provides an optional "Verify On-Chain" feature accessible from the decrypted file view. This feature:

  • Fetches the transaction record from the Solana blockchain using the TxID embedded in the file.
  • Compares the on-chain hash prefix (first 16 hex characters of SHA-256) with a locally computed hash of the encrypted file content.
  • Returns one of the following results: Match (integrity confirmed), Mismatch (file may be altered), No Memo Found, Transaction Not Found, or Error.
  • Requires an active internet connection and is subject to Solana RPC availability. VaultMemo uses a three-endpoint fallback chain: rpc.ankr.com/solana (primary), api.mainnet-beta.solana.com (secondary), mainnet.helius-rpc.com (last resort).
  • Verification is free of charge. Solana network fees do not apply to read-only RPC queries.
Limitation: The on-chain record is a cryptographic proof only and does not constitute legal verification, notarization, or legally admissible evidence in any jurisdiction.

Encryption Certificate

VaultMemo v1.2 allows users to generate a shareable encryption certificate image after saving or decrypting a memo. The certificate:

  • Is generated entirely on-device as a PNG image.
  • Contains: file name, truncated wallet address, truncated TxID, encryption tier, algorithm, byte size, and timestamp.
  • Does not contain your memo content, full wallet address, or private keys.
  • Is free of charge — no SKR payment is required.
  • Can be shared via the Android system share sheet to any app (messaging, email, cloud storage, etc.).
  • Available in English (default) with a toggle to the app's current language.
Disclaimer: The certificate is an informational document only and does not constitute legal proof of encryption, authorship, or timestamp beyond the cryptographic record on the Solana blockchain.

Storage Permissions

VaultMemo saves encrypted .vam files to two locations when the user taps "Encrypt & Save":

  • DocumentDir/VaultMemo/ — app-internal storage for the in-app file list (cleared on app uninstall)
  • Downloads/VaultMemo/ via MediaStore — permanent storage accessible via the file manager (persists after app uninstall)

No background writes occur. All file operations are user-initiated.

Data Loss Disclaimer

The developer is not liable for data loss resulting from: device failure; app uninstallation (note: .vam files in Downloads/VaultMemo/ persist and can be re-imported; files in DocumentDir/VaultMemo/ are deleted on uninstall); loss or compromise of the wallet seed phrase; user deletion of backup files; or in-app deletion of files from the file list. Maintain backups in multiple locations.
